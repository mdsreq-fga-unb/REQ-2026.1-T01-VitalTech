## Entregas Apresentadas

### Levantamento de Requisitos
* Apresentação dos requisitos funcionais e não funcionais mapeados por meio de conversas (WhatsApp e reuniões) e análise de documentos (formulários físicos atuais fornecidos pelo Marcelo).
* **Atenção:** Alguns requisitos ainda dependem de validação final com o cliente devido à ausência na reunião.

### Matriz de Priorização (Valor vs. Complexidade Técnica)
* Estruturada utilizando uma escala Likert (1 a 5) para priorizar as necessidades principais.
* Quadrantes de baixo impacto x alta complexidade foram deixados vazios propositalmente (padrão de mercado para descartar funcionalidades inviáveis).

### Prototipação de Alta Fidelidade (Figma)
Foco em uma interface simples e intuitiva para os cuidadores, evitando retrabalho no preenchimento diário.

**Telas Demonstradas:**

* **Login:** Acesso simples (e-mail e senha) para cuidadores e stakeholders.
* **Gestão de Usuários:** Tela para administradores cadastrarem novos perfis de profissionais.
* **Painel de Residentes:** Listagem dos 74 idosos cadastrados, com filtros de busca por nome ou número do quarto.
* **Prontuário/Resumo:** Histórico do paciente com filtro de datas, exibindo sinais vitais, alimentação, hidratação, higiene, eliminações e ocorrências.
* **Adição de Evolução:** Menus de rápido preenchimento e interface simplificada para registro de rotinas (sinais vitais, banho, etc.).
* **Administração de Medicamentos:** Área para associar e registrar a medicação de cada paciente.
* **Modo Offline:** Tela de referência demonstrando que o sistema salvará os dados localmente em caso de queda de internet.
