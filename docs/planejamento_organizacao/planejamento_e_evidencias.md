# Quadro de Planejamento

Este quadro centraliza o cronograma do projeto, o status real de execução e as evidências associadas às sprints. A organização segue o que foi definido para a aba **Planejamento e Organização**: apresentar o planejamento com evidências, sem substituir a rastreabilidade detalhada dos documentos de Visão, Requisitos, Story Map, DoR/DoD e atas das sprints.

Considerando a estratégia ScrumXP, iterativa e incremental, o cronograma foi estruturado em sprints com entregas funcionais associadas às User Stories planejadas no Story Map do projeto. Cada sprint busca entregar uma fatia funcional do sistema que possa ser demonstrada, verificada pela equipe e validada com o cliente, evitando a organização das entregas apenas por camadas técnicas isoladas.

O sequenciamento das sprints segue a jornada principal do usuário no sistema, partindo da base de acesso e cadastros iniciais, avançando para registros assistenciais, consulta de histórico, governança de acesso e visão consolidada do residente.

Em todas as sprints, as atividades de Engenharia de Requisitos ocorrem de forma contínua, incluindo refinamento das User Stories planejadas, revisão dos critérios de aceitação, atualização da rastreabilidade, aplicação de DoR e DoD, validação com o cliente e revisão dos Requisitos Não Funcionais aplicáveis.

Para apoiar as evidências de execução, cada sprint mantém seus registros nos artefatos correspondentes de Planning, Dailys, Review e Retrospectiva. Esses artefatos servem como referência para demonstrar o que foi planejado, executado, validado e ajustado ao longo do projeto.

> **Nota sobre a cadência:** embora o ciclo de desenvolvimento seja pautado em sprints quinzenais, a Sprint 0 correspondeu a uma fase estendida de concepção e alinhamento. Esse período foi necessário para compreender o domínio do Lar dos Velhinhos, negociar o escopo inicial com a organização parceira e preparar a equipe para as sprints de execução.
