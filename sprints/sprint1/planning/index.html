    Sprint 1 - Planning

    Período

    03/05/2026 a 16/05/2026.

    Objetivo da Sprint

    Realizar o levantamento inicial de requisitos por Brainstorm e estruturar o DoR do produto VitalTech, preparando os artefatos necessários para as próximas sprints de execução.

    Escopo Planejado

    Frente Atividades planejadas Evidência relacionada
    Brainstorm Levantar e discutir funcionalidades iniciais, problemas percebidos e pontos de refinamento para os requisitos. docs/sprints/sprint1/dailys.md, docs/sprints/sprint1/review.md
    Requisitos e User Stories Revisar RFs, RNFs e primeiras User Stories geradas a partir do refinamento da sprint. docs/visao/requisitos.md, docs/visao/user_stories.md
    Definition of Ready Definir critérios de entrada para que as User Stories possam ser consideradas prontas para sprint. docs/visao/dor_dod.md
    Story Map Organizar a jornada principal do usuário e distribuir User Stories em sprints. docs/visao/story_map.md
    Organização do processo Registrar dailys assíncronas, revisar entregas e conduzir review e retrospectiva. docs/sprints/sprint1/dailys.md, docs/sprints/sprint1/review.md, docs/sprints/sprint1/retrospectiva.md

    Atividades de Engenharia de Software

    • Organizar os ritos da Sprint 1 no formato adaptado do ScrumXP.
    • Consolidar evidências de acompanhamento por meio de dailys assíncronas.
    • Revisar os artefatos produzidos antes da entrega da unidade.
    • Registrar pontos de melhoria na retrospectiva para orientar as próximas sprints.

    Atividades de Engenharia de Requisitos

    • Realizar Brainstorm para levantar e refinar funcionalidades iniciais.
    • Refinar a lista de requisitos funcionais e não funcionais a partir das discussões registradas.
    • Revisar critérios de aceitação associados às User Stories.
    • Definir e aplicar o DoR inicial às histórias preparadas para as próximas sprints.

    Critérios de Conclusão da Sprint

    • Brainstorm registrado nos artefatos de acompanhamento.
    • Backlog inicial estruturado.
    • User Stories documentadas com critérios de aceitação.
    • RFs e RNFs revisados e organizados.
    • DoR inicial definido para apoiar as próximas sprints.
    • Review e retrospectiva registradas.
